# The Import Model


程序由一行行的代码构成。代码中含有各式各样的符号：分隔符（空格、行分隔符和块分隔符等）、保留字（reserved word）、操作符（operator）和用户自定义符号。一个简单的程序极有可能通过自身就能完成大部分任务，比如你在 LeetCode 上提交的代码，可以通过一个单一入口解决非常复杂的问题。但是绝大部分情况下，一个完整的程序是不同模块之间协作完成的。

模块（module）是个复杂的概念。这个词在不同的编程语言中表示的是完全不同的意思，甚至在同一个编程语言中，也有各种不同类型的模块（比如JavaScript的CommonJS module、AMD module 和 ECMAScript module）。在脱离了实际编程语言背景的情况下，本文中所称的模块，表示的是用于构建或者导入的代码单元。如一个 Java 源文件，可以构建成一个或者多个类，能被其他 java 文件引用。同理也适用于 python 源文件之于 pyc 二进制文件，TypeScript 源文件之于 JavaScript 目标文件。

进行这种划分的原因就在于，每次的 import（或者 use / require / include / open）动作都是在源文件中完成的。有了这个相对统一的划分标准，我们就能够考察不同编程语言其在引用模型上的区别。进一步地，我们也能对比出为什么不同编程语言把 module 这个概念搞出这么多变种。
